“AI安全日志分析助手”

## 🧭 整体学习路径

1. **搭建基础环境**（Ubuntu 22.04 + Python + Docker）
2. **配置开发工具**（VS Code + 插件）
3. **第一个Agent脚本**（直接调用LLM分析日志）
4. **引入LangChain构建Agent**（使用工具和推理）
5. **增加RAG能力**（向量数据库 + 安全知识库）
6. **多Agent协作**（CrewAI或LangGraph）
7. **项目打包与部署**（FastAPI + Docker）

### **📂 核心文件深度解读**

#### **1.** `tools/log_tools.py` **(基础工具层)**

- **定位**：**IO 操作层**。

- **功能**：负责最基础的文件读取、大小检查、格式验证。

- 特点：

  - 它是 `simple_analyzer.py` 和 `api.py` 目前直接依赖的工具。
  - **无智能**：它只是把文件内容读出来变成字符串，不做任何分析。
  - **作用**：为上层 Agent 提供“眼睛”，让它们能看到日志内容。

#### **2.** `tools/retrieval_tools.py` **(RAG 检索层 - 关键！)**

- **定位**：**知识库连接器**。

- **功能**：连接向量数据库（ChromaDB），实现语义搜索。

- 核心逻辑：

  - 接收一个查询（例如：“查找所有 SSH 暴力破解尝试”）。
  - 将查询转化为向量（Embedding）。
  - 在 `chroma_db` 中搜索最相似的日志片段。
  - 返回相关的上下文片段给 LLM。

- **价值**：解决了 LLM **上下文窗口有限**的问题。即使日志有 1GB，也可以通过检索只把“最可疑的 500 行”发给 LLM，既省钱又精准。

#### **3.** `rag/build_vector_db.py` **(数据预处理层)**

- **定位**：**知识库构建器**。

- **功能**：离线处理历史日志，建立索引。

- 流程：

  1. 读取原始日志文件。
  2. **分块 (Chunking)**：将长日志切分成小段（例如每 500 行一块），保留重叠部分以防切断上下文。
  3. **向量化 (Embedding)**：调用 Embedding 模型将文本块转为向量。
  4. **存储**：存入 `chroma_db` 目录。

- **使用场景**：当你有一堆历史日志（如过去一个月的 `auth.log`），运行此脚本后，Agent 就能“回忆”起过去的攻击模式。

#### **4.** `agents/log_agent.py` **(单兵作战升级版)**

- **定位**：**LangChain 单 Agent 封装**。

- **功能**：比 `simple_analyzer.py` 更结构化。

- 区别：

  - `simple_analyzer.py` 是过程式代码（直接调 API）。
  - `log_agent.py` 使用了 LangChain 的 `AgentExecutor` 架构。
  - 它可能挂载了 `log_tools` (读文件) 和 `retrieval_tools` (查库) 两个工具。
  - **能力**：LLM 可以自主决定：“我需要先读一下文件，发现不懂的攻击特征，再去向量库里查查以前有没有类似的”。

#### **5.**  `crew_agents_stream_mapreduce.py` **(多智能体协作层)**

- **定位**：**CrewAI 团队编排**。

- **功能**：定义多个不同角色的 Agent 协同工作。

- 角色示例：

  - **Log Parser (解析员)**：负责清洗日志，提取 IP、时间、错误码。
  - **Threat Hunter (威胁猎手)**：利用 `retrieval_tools` 检索已知攻击模式，进行深度分析。
  - **Reporter (报告员)**：将前两者的发现汇总，写成人类可读的报告。

- `stream_mapreduce` 的含义

  ：这是一种处理长文本的高级策略。

  - **Map**：把大日志切分，分发给多个 Agent 并行分析。
  - **Reduce**：最后由一个总控 Agent 汇总所有结果。
  - **目的**：解决超长日志分析慢、容易丢失细节的问题。

#### **6.** `main_prod_chunked.py` **(早期版集成入口)**

- **定位**：**早期的生产尝试版**。
- **状态**：这个文件试图把 FastAPI、CrewAI、分块逻辑全部写在一个文件里，有些“混乱”。
